1
Research & Discovery
Weeks 1-2 (Feb 23 - Mar 9)

UX Research & Problem Definition

  • User Pain Point Analysis: Identified that users struggle with complex insurance terminology and scattered information
  • Competitive Research: Analyzed Benefits Oracle & United Healthcare user experiences
  • Technology Research: Investigated voice UI patterns and conversational design principles
  • Stakeholder Interviews: Collaborated with Spectacle Health CEO to understand business goals

Key UX Insights Uncovered

  • Cognitive Load: Insurance decisions overwhelm users with too many options
  • Trust Factor: Users need confidence in AI recommendations without legal liability
  • Accessibility: Conversational interface reduces barriers for non-tech-savvy users
  • Efficiency: Users want quick, specific answers to coverage questions
2
Empathize & Define
Weeks 2-3 (Mar 9 - Mar 16)

User Research Insights

  • Users struggle with complex insurance terminology
  • Need for specific medication coverage questions
  • Desire for recommendation vs. redirection to websites
  • Risk management concerns for definitive answers

Target Questions Identified

  • "Will this insurance cover my diabetes medication?"
  • "Do I qualify for weight loss medication coverage?"
  • "What type of insulin is included in this plan?"
Challenge: Balancing helpful guidance with legal/medical accuracy requirements
3
Technical Foundation
Weeks 3-4 (Mar 16 - Mar 30)

Vector Database Development

  • Set up Weaviate Embedded instances
  • Processed Exchange Rate PDF and Uniform Glossary PDF
  • Implemented text extraction using pdfplumber
  • Created schema for diabetes and weight loss medications

Data Processing & Querying

  • Developed text vectorization using HuggingFace models
  • Tested query responses with healthcare-specific questions
  • Established relationships between multiple databases
Technical Challenge: Jumbled output initially - only pulling single words instead of coherent responses
4
Design & Prototype
Weeks 4-5 (Mar 30 - Apr 13)

Low-Fidelity User Flow Design

  • Registration Flow: Designed optional account creation vs. guest access pathway
  • Main Chat Interface: Created dual input system (text input + AI suggested prompts)
  • Plan Suggestions Flow: Mapped journey from user query to final insurance plan summary
  • Settings Integration: Designed profile, exit, settings, and home navigation

High-Fidelity Mobile Interface

  • AI Chat Landing: Clean header with hamburger menu and settings dropdown
  • Conversation UI: Question input field with suggested actions below
  • Smart Prompts: "Compare two plans", "List popular plan", "Filter by budget" options
  • Message Interface: Clean "Message Ai" input with send functionality

Key Interface Decisions

  • Prioritized mobile-first design for accessibility
  • Integrated suggested prompts to guide user conversation
  • Maintained consistent navigation patterns throughout
  • Designed for both guided and free-form interaction styles
Success: Created seamless flow from registration to complex insurance queries with intuitive mobile interface
5
Refinement & Presentation
Weeks 5-6 (Apr 13 - Apr 28)

Final Prototype Features

  • Functional user input with typing feature
  • AI response generation and text population
  • Smooth transitions between chat screens
  • Submit button and interaction feedback

Design Showcase Preparation

  • Created comprehensive presentation covering full process
  • Documented challenges, successes, and future steps
  • Prepared live demonstration of working prototype
  • Presented to stakeholders and Design Interactive community
Achievement: Final prototype looked polished in presentation mode with professional UI consistency
